# 当前回归基线

## 保护目标

当前基线保护的是已经跑通的「上传四视图 -> 生成/修改剧情 -> 生成 1 版 3 张分镜 -> 编译视频执行包 -> 用户手动点击生成视频」主链路。

重点保护以下行为：

- 前端只展示 1 版 3 张独立分镜图，每张图可以点开查看大图。
- 视频执行包编译后不自动提交视频，必须由用户手动点击生成视频。
- 视频生成前仍会经过安全预检和 provider-specific visual harness。
- 前端不直接展示最终视频 prompt、harness 原始字段、模型限制对象、后台 prompt preview 等内部诊断内容。
- 分镜和视频等待过程必须有可见动效，不能退回静态等待画面。

## 固化命令

```bash
npm run test:baseline
```

这个命令会先构建项目，再运行静态基线检查和浏览器流程检查；任一环节失败都会以非零状态退出。

## 后续修改规则

凡是改动 UI、分镜生成、视频执行包、provider harness、视频提交或进度展示，必须在改动前后各运行一次：

```bash
npm run test:baseline
```

如果需要引入 `shadcn/ui` 或替换大块前端组件，先保持这条基线通过，再逐个模块替换；不要在同一次修改里同时迁移 UI 基建和改变视频业务链路。
