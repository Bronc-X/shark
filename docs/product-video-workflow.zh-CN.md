# 产品一致性优先的分镜图生视频工作流

## 目标

本项目只服务一个核心目标：为五个固定可穿戴充气服产品生成短视频时，先保证“货对版”，再让动作、道具互动和小包袱真正被演出来。

默认平衡顺序为：

1. 产品一致性
2. 视频完成度：动作段落、道具互动和小包袱必须可读
3. 场景清晰
4. 时长

如果动作会导致产品漂移，必须把动作改写成最近的安全可见版本，而不是直接忽略动作，也不能把视频压缩成几乎不动的摆拍。

## 当前范围

当前长期预设品类是五个可穿戴充气服：

- 鲨鱼充气服
- 奶牛充气服
- 灰色老鼠充气服
- 青蛙充气服
- 相扑充气服

因为产品集合固定，当前版本不收集商品卖点、价格、SKU、折扣、渠道 CTA、字幕、举牌、提示语、画面文字或后期销售文案。当前五个产品也没有足够好的参考视频，所以不做参考视频复刻。

## 用户可见流程

界面只露出 3 个步骤：

```text
上传四视图 -> 剧情意图与候选分镜 -> 生成视频
```

“定款/产品锁定”是后台自动步骤，不作为用户可点击步骤露出，也不要求用户逐条确认。昂贵的视频生成必须由通过预检的视频执行包触发；执行包来自已确认剧情意图、入选分镜、产品锁点和动作模式。

## 1. 上传四视图

上传阶段必须先提供四张平行的核心视图，没有主图和参考图之分：

1. 正面图：锁定正面比例、脸窗/面部区域、拉链、脚部和正面装饰。
2. 左侧图：锁定左侧厚度、左侧轮廓、左侧可见结构，以及该侧缺失的结构。
3. 右侧图：锁定右侧厚度、鼓风阀方向和高度、右侧附加结构、侧缝、尾部边缘和鞋底。
4. 背面图：锁定背部主体、背部竖缝、尾鳍/背部附加结构和背面轮廓。

用户不再上传细节图。长期产品可以在本地预设中保存后台辅助角度，用来强化阀门、尾巴、围巾、腰带、拉链、缝线、褶皱或材质等易漂移部位。

上传规则：

- 正面、左侧、右侧、背面四张核心图全部上传后，才允许进入剧情和分镜。
- 上传正面图后不能自动跳到后续步骤。
- 不接受只上传一张图或只上传正面图。
- 不把视频作为必需输入；当前产品一致性策略以四张静态图片为准。
- 前端提交的核心字段是 `image_urls`，且必须包含 4 张可读核心图片。
- `image_urls` 的语义顺序固定：`image_urls[0]` 是正面，`image_urls[1]` 是左侧，`image_urls[2]` 是右侧，`image_urls[3]` 是背面。
- 本地预设辅助角度提交到 `support_image_urls`，由系统自动附带，不作为用户上传入口。
- 后端必须拒绝 `blob:` 本地预览地址，只接受 `data:image/` 或 `http(s)` 图片地址。

## 2. 后台自动定款

四视图上传完成后，系统自动生成产品一致性合同。这个合同不暴露成用户步骤，但必须进入剧情意图、分镜和视频执行包。

一致性合同包含：

- 品类级锁定：始终是可穿戴充气服。
- 四视图拓扑：正面、左侧、右侧、背面各自约束真实物理位置。
- 易漂移部位：阀门、透明脸窗、拉链、尾鳍、鳃线、鞋子、褶皱、接缝；本地预设辅助角度只作为补充证据强化这些点。
- 禁止变化：不得重绘、平均、拼贴、挪位、复制、删减、放大、缩小、风格化产品、添加文字、logo 或新配饰。
- 人体体型包络：产品必须像真人穿着一件轻到中度充气服，不能变成巨大圆顶头、桶状身体、站立气球、展示道具或重新设计的吉祥物。
- 体积包络：保持四视图共同的尺寸、比例、厚度和中低充气轮廓。

四视图不是拼贴素材，也不是装饰参考。它们是同一个产品的拓扑地图。

## 3. 剧情 / 动作意图

系统先调用大模型同步生成或改写剧情/动作意图。用户可以留空让模型自动生成，也可以提供一句方向，再用模型继续修改、重写或细化。

剧情意图必须输出：

- 一个短标题。
- 一个连贯的动作前提。
- 一个场景锚点。
- 三到五个可读动作节点。
- 产品风险提示。

剧情意图不得包含商品卖点、价格、SKU、折扣、渠道 CTA、字幕、举牌、提示语、画面文字或后期销售文案。

## 4. 分镜生成

分镜根据已确认剧情意图和四视图产品锁点生成。分镜可以多次生成和筛选，因为图片成本低，应该在这里尽量收敛掉动作、镜头和产品一致性的不确定性。

分镜规则：

- 基于同一个剧情意图生成多张候选。
- 入选分镜至少 3 张，最多 5 张。
- 每张入选分镜必须属于同一产品、同一场景和同一剧情。
- 每张分镜必须带动作说明、视角说明和检查结果。
- 看不见的侧面或背面细节不能被挪到当前视角，除非镜头物理上能看到该表面。
- 不得在场景或产品上新增字幕、招牌、标签、价格、CTA、logo、贴纸或任何可读文字。

## 5. 生成前预检与执行包

检查必须发生在视频生成前，而不是视频生成后的假质检。

预检内容：

- 入选分镜数量是否有效。
- 是否存在 `fail` 分镜。
- 分镜是否共享同一产品、同一场景和同一动作路径。
- 镜头路径是否符合四视图拓扑。
- 提示词和分镜是否包含文字、卖点、CTA、价格、logo 或举牌需求。
- 是否有明确的起始分镜和结束分镜。

只有预检通过，系统才编译 `video_execution_package`。执行包必须包含入选分镜图、剧情意图、产品锁点、动作模式、镜头路径、最终视频 prompt 和风险备注。

## 6. 生成视频

视频模型一次性接收执行包。生成视频成本高，所以分镜生成本身不能触发 `/api/video`。

视频请求必须继承：

- 入选分镜起始图。
- 四张核心视图产品硬锁。
- 可穿戴充气服品类锁。
- 视图拓扑锁。
- 体积包络锁。
- 剧情意图和入选动作顺序。
- 预检后的镜头路径和动作模式。

如果动作与产品一致性冲突，保留产品外壳、比例和部件归属，同时把动作调整为最近的安全可见版本；不能直接忽略动作，也不能只生成几乎静止的视频。

## 鲨鱼充气服默认锁定点

正面：

- 白色腹部区域。
- 横向透明脸窗。
- 脸窗下方垂直拉链。
- 蓝色外边。
- 白色内侧手鳍。
- 蓝色脚套和黑鞋。
- 人体穿戴尺度：外壳只比真人肩宽和躯干略宽，头部和躯干不能放大成圆顶吉祥物或桶状气球。

左侧 / 右侧：

- 只能有 1 个黑色圆眼。
- 必须有 5 条黑色弧形鳃线。
- 橙色圆形鼓风阀必须在实际所在侧的侧腰，方向和高度正确。
- 左右侧鳍、侧缝、鞋底、侧面厚度和不对称关系必须稳定。

背面：

- 纯蓝背部。
- 中轴背部竖缝。
- 中轴尾鳍。
- 背部轮廓不能膨胀成无结构圆柱。

本地辅助角度 / 局部证据：

- 布料褶皱。
- 接缝张力。
- 阀门网格。
- 脸窗透明反光。
- 拉链和边缘缝线。

## 工程实现约束

前端：

- 三个步骤：上传、分镜、视频。
- 上传区必须是四张平行卡片。
- 正面、左侧、右侧、背面四张核心图没齐时，“素材上传完成”禁用。
- 用户界面不提供细节图上传入口；长期产品的辅助角度由本地预设自动附带。
- 图片生成和文字提示词生成 API 由后端固定配置，用户界面不能要求用户填写图片/文字 API Key 或 base URL。
- 剧情意图界面必须调用大模型，并支持大模型改写。
- 分镜界面可以在提交视频前多次生成候选。
- 视频页展示执行包生成的只读最终视频 prompt。
- 换任意产品图、产品类型、剧情意图、入选分镜、场景、动作模式或比例后，旧分镜、旧执行包、视频和状态必须作废。
- 用户看不到“定款”步骤。
- 用户看不到技术 URL 字段。
- 图片预览必须 `object-fit: contain`，不能裁切产品。

后端：

- 图片生成和文字提示词生成统一使用后端 `IMAGE_TEXT_BASE_URL` / `IMAGE_TEXT_API_KEY`，不得把这组凭证传到浏览器表单。
- `/api/product-locks` 必须校验 `image_urls` 正好 4 张核心图，并返回结构化产品锁。
- `/api/story-intent` 负责生成或改写剧情/动作意图，不能输出卖点或文字覆盖需求。
- `/api/storyboards` 必须在四视图、剧情意图和产品锁点齐备后才生成候选分镜。
- `/api/video-package` 负责分镜预检并返回视频执行包。
- `/api/video` 必须拒绝没有通过 `video_execution_package` 的请求。
- 不再接受 `foreground_source_url` 作为输入。
- prompt 必须明确“四视图是拓扑地图，不是拼贴要求”。
- prompt 必须明确“不要平均四张图生成新产品”。
- prompt 必须明确“不要把物理不可见的侧面/背面细节强行挪进当前视角”。
- 视频 prompt 必须继承入选分镜路径和四视图产品锁。

验证：

- 每次改工作流必须运行 `npm run test:baseline`。
- 必须浏览器检查上传 -> 剧情意图 -> 分镜 -> 执行包 -> 显式视频提交路径。
- 必须确认 `/api/video` 不会在 `/api/video-package` 和 `/api/video-safety` 通过前提交。
- 必须检查五个本地产品预设、四张核心平行上传卡、无细节图上传区、禁用状态、隐藏定款步骤、隐藏 URL 字段、无横向溢出。
